# AGENTS.md

Guidance for cloud agents working in this repository. See `CLAUDE.md` for full architecture and commands.

## Cursor Cloud specific instructions

### What runs where

| Command | Port | Serves | Use when |
|---------|------|--------|----------|
| `node server.js` | 3400 | Project **root** (source HTML, not `dist/`) | Quick static page edits |
| `npx netlify-cli dev --offline` | 8888 | Static + `/api/*` Netlify functions | Booking engine, quotes, guest app APIs |
| Playwright E2E (auto) | 3401 | `dist/` via test helper | `npm run test:e2e` only |

**Netlify CLI is not a project dependency** — use `npx netlify-cli dev` (add `--offline` to skip Netlify account linking). Without credentials in `.env`, functions return mock OTASync data (`isMock: true`); that is enough for the booking flow through the Wompi payment step.

### Standard validation

```bash
npm ci
npx playwright install --with-deps chromium   # first run / fresh VM only
npm test                                    # build + 58 unit tests + 27 e2e (1 skipped)
```

There is no separate lint script; `npm run build` validates i18n key parity and structural checks in unit tests.

### Local env file

Copy `.env.example` → `.env` only when testing real OTASync, Wompi sandbox, Resend, or Netlify Blobs (quotes). Mock mode works without `.env` under `netlify dev`.

### Dev workflow notes

- Edit sources in the repo root; `dist/` is generated by `npm run build`.
- `npm run dev` watches only `motor-app.jsx` → `motor-app.js` (booking engine).
- Bilingual changes require both `.lang-es` / `.lang-en` in HTML and `es` / `en` keys in `motor-app.jsx` `i18nEngine`.
- Do not commit changes under `dist/` unless the project explicitly requires it (CI rebuilds on every test run).

### Services for manual E2E

1. `npm run build` (optional for `server.js`; required before deploy)
2. `npx netlify-cli dev --offline` for full booking/quote/guest API flows
3. `node server.js` if only marketing pages need checking

Guest app demo mode works without OTASync when `GUEST_APP_DEMO_MODE` is unset in local non-production (see unit tests).
